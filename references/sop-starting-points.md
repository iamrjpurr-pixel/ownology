## Winemaker Magazine — Garagiste 2023 SOP slides (pages 1-9 findings)
Status: Foundational reference source. Treat as Ownology's baseline winemaking SOP philosophy and structure guide.

### Core definitions and framing
From the visually readable slides, the presentation defines SOPs as a set of instructions to carry out routine tasks. A well-written SOP should answer five operational questions: **who, what, where, when, and hazards**.

The presentation distinguishes SOPs from recipes by emphasizing winery variability. The key reasons given are **vintage variability**, **varietal variability**, **fermentation variability**, and the fact that **each wine lot is different**. This is important for Ownology because it supports a SOP architecture that is principle-based and decision-aware, rather than rigidly recipe-like.

### Why SOPs matter
The slides frame SOPs as especially important when moving from small-scale to commercial winemaking. The stated benefits are to **increase quality**, **avoid pitfalls**, support **employee training**, and address **Food Safety Modernization Act (FSMA)** expectations.

A dedicated FSMA slide groups winery SOPs into three pillars:

| Pillar | Supporting documents or purpose |
|---|---|
| Food safety operating practices | Policies and logs |
| Operation of machinery | Instructional documents also used for training |
| Winemaking processes | Quality control and process analysis/logs |

### Suggested approach to writing SOPs
The slides present a three-level approach:

| Level | Description |
|---|---|
| Generalized outline | A high-level sequence of winemaking steps |
| Style-level outline | Separate outlines for whites, reds, and rosé |
| Wine-specific SOP set | Detailed SOPs for each individual wine planned |

The "Where to start" and follow-up slides recommend beginning with a **generalized outline of winemaking** built around these stages: **starting ingredient, fermentation, post-fermentation or aging, and bottle**.

For a style-based SOP set, the slides recommend the following sequence for **white, rosé, or red wine**:
- Starting ingredient
- Pre-fermentation
- Fermentation
- Skin contact
- Post-fermentation / aging
- Filtering
- Bottle

For an **individual wine** SOP set, the slides recommend this structure:
- Style of wine
- Source of starting ingredient / harvest
- Pre-fermentation
- Fermentation
- Post-fermentation / aging
- Blending, fining, stabilities
- Bottling

### Implication for Ownology SOP design
Ownology SOPs should be organized as a layered system:
1. General winery SOP framework
2. Style-specific SOP families (white, rosé, red, sparkling, fortified where relevant)
3. Wine-specific SOP instances for actual production lots or SKUs

This source should be treated as the conceptual "bible" for SOP structure, not as a compliance source. Compliance requirements should still be attached from primary sources such as Wine Australia, FSANZ, Safe Work Australia, and state regulators.

Source: Jenne Baldwin-Eaton, "Improving Winery SOP's," Garagiste 2023 / Winemaker Magazine PDF provided by user.

## Winemaker Magazine — Garagiste 2023 SOP slides (pages 10-14 findings)

### Operational considerations added in this section
The slides add several practical dimensions that should shape SOP design beyond the basic process flow.

| Topic | Guidance captured from slides |
|---|---|
| Equipment | More expensive equipment, more electronics, higher voltage needs, and increased maintenance all increase the need for documented SOPs |
| Style of wine | SOP design should branch by white, orange, rosé, or red; fruit-forward vs. oak style; dry/off-dry/semi-sweet/dessert/port; quick release vs. reserve; sparkling; and cultivar-driven style decisions |
| Source of starting ingredient | SOPs should distinguish between vinifera, hybrid, fruit, and honey starting materials |

### Harvest considerations
The harvest slide identifies the following elements that should be explicitly captured in a harvest SOP:

| Harvest factor | Notes |
|---|---|
| Picking parameters | Should match intended style of wine |
| Harvest method | Hand-picked or mechanical |
| Transport | Transportation or shipping arrangements |
| Storage timing | Storage in cooler or straight from vineyard |
| Handling vessels | Bins, lugs, or buckets |
| Fruit condition | Whether fruit was previously frozen |

### Pre-fermentation module
The pre-fermentation slide identifies a concrete checklist for SOP drafting:

| Pre-fermentation element | Included in slide |
|---|---|
| Additions | Enzyme and/or tannin addition |
| Contact regime | Skin contact (cold soak) |
| Maceration approach | Carbonic maceration |
| Lab work | Lab analysis |
| Protection | Sulfur dioxide / bioprotectant |

### Implication for Ownology SOP design
This section reinforces that SOPs should not just describe process sequence. They should also encode the winery's decision variables: equipment profile, wine style, source material, harvest logistics, and pre-fermentation options. These variables can become a template schema for future Ownology SOP generation.

## Winemaker Magazine — Garagiste 2023 SOP slides (pages 15-19 findings)

### Fermentation module
The fermentation slide adds a practical set of decision points that belong in a wine SOP:

| Fermentation element | Guidance captured from slide |
|---|---|
| Yeast strategy | Natural, non-saccharomyces, or Saccharomyces |
| Yeast handling | Proper rehydration, temperature range, and nitrogen requirements |
| Contact regime | Skin contact |
| Secondary fermentation | ML fermentation |
| Vessel / maturation input | Oak |
| Monitoring | Lab analysis, including tracking Brix and Clinitest |

### Post-fermentation / aging module
The post-fermentation slide identifies the main aging-stage control points:

| Post-fermentation element | Guidance captured from slide |
|---|---|
| Extraction / phenolic choice | Extended maceration |
| Lees work | Sur lie / batonnage |
| Maturation input | Oak |
| Protection | Oxidation prevention |
| Stability / hygiene | Microbial protocols |
| Monitoring | Lab analysis |

### Bottling module
The bottling slide presents bottling as a sequence rather than a single step. The order shown is:
1. Blending
2. Fining
3. Stabilities
4. Filtering
5. Bottle

This is useful because it turns the end-of-process stage into a controlled pre-bottling workflow rather than a single packaging event.

### Summary philosophy from the final slide
The closing summary is strategically important for how Ownology should frame future SOPs. The presentation states that improving SOPs is an **evolving process**. It describes SOPs as a **roadmap** that keeps the winery on track for success while also warning about possibilities, and explicitly says **we are not seeking perfection**. It also emphasizes that SOPs are a **combination of knowledge and experience**, and that learning often happens when things go wrong.

### Implication for Ownology SOP design
Ownology SOPs should be framed as living operational systems, not static binders. The best product expression would likely include:

| Design principle | Implication for Ownology |
|---|---|
| SOPs evolve | Versioned SOPs with review dates and revision history |
| SOPs are roadmaps | Decision-aware guidance, not rigid scripts |
| Knowledge + experience | Blend regulatory rules, technical best practice, and winery-specific notes |
| Learning from failure | Incident notes, exceptions, and lessons learned should feed SOP revisions |

### Working position for future SOP work
Treat this Winemaker Magazine deck as the **foundational conceptual source** for SOP architecture and tone. It should guide how Ownology structures winery SOPs: process-based, stage-based, style-aware, and iterative. It is not the legal authority for compliance limits, but it is a strong practical authority for how a winery should think about SOP creation.
